In this project, we will study a bit of CSS by making some boxes with certain properties. To see what the final version should look like, check out the finished product.

  • Learn a few CSS properties
  • A computer to use for a bit
  • Basic familiarity with html
  • Desire to add custom style to a website

Let's set up an HTML file complete with a head and a body. You should already know how to do this, so this step should be very easy. If you are unsure of how to go about this, I suggest you try out the HTML basics code project.

<!DOCTYPE html>

<html>
  <head>
  </head>
  <body>
  </body>
</html>

Firstly, we are going to want to create a new file called style.css (it doesn't have to be named style, but that's a good convention, so let's stick with it). Inside this file is where we will put all of our CSS rules. The problem is, how do we tell the HTML file that we're going to be using a style sheet? For this, we have the link tag.

In the head of your html file, let's go ahead and put a link tag with three attributes. The first attribute is called type and is used to specify the type of file we're linking. In order to tell our document that we're linking a text css file, we set the value to "text/css". The second attribute is called rel and we set that value to "stylesheet". The third value tells the document where it can find the stylesheet we're linking and is called href with the value being the path to the CSS file. Go ahead and link your CSS file. In the next step, we'll make sure your link is working correctly.

<html>
  <head>
    <link type='text/css' rel='stylesheet' href='/css/lecture2.css'>
  </head>
  <body>
  </body>
</html>

Since the background color extends over the entire body (regarless of the size of the screen, etc), we want to set the background color of our body. To set the backroung property of an HTML tag, we use this syntax:

tag {
  property1: value1;
  property2: tag2;
  ...
}

So, to set the background-color property of our body, our code should look like:

body {
  background-color: color-choice;
}

In CSS colors can be specified via their color name (if CSS knows about this color), their unicode index, their rgb (red/green/blue) value, or even their rgba (where A denotes the opacity with 1 being fully opaque and 0 being fully transparent). In our case, we want the rgb value to be 40, 40, 40. To do this, we can set the background-color property to rgb(40, 40, 40). Go ahead and do that now.

body {
  background-color: rgb(40, 40, 40);
}

Firstly, for these squares, we see that we want a repeated style. By that I mean that we want four objects with exactly the same style. This is a good time to use a means of abstraction that CSS has built in that lets us style objects with certain classes. The syntax for this is basically the same as for tags except we start the class name with a period (to specify that we're talking about classes and not something else). For example, to give everything with class class-name prop1 set to attr1, etc, the code would look like:

.class-name {
  prop1: attr1;
  prop2: attr2;
  ...
}

Being as how we have 4 boxes with much of the same style, we might do well to utilize this tool. So let's get on that! What should we name our class? It should be something descriptive of what it does. How about box? Let's use that. In your HTML code, create four div elements with their class attributes set to "box".

Great! Now we can style by class and save ourselves a lot of time! The first thing we want to specify is a background-color so that we can see our elements at least (in truth there's a better way than that, but that's out of the scope of this project). Let's set the color to an rgb value of 40 red, 200 green, and 60 blue.

.box {
  background-color: rgb(40, 200, 60);
}

Great! We've got the background color set, but we still don't see anything! What the French toast?! Well, we haven't put anything inside the divs yet, so there's noheight to them yet! Luck for us we can set the height property manually with CSS! Inside the style for the box class, let's add a height property, and set it to 100px. This time do not put the value in quotes.

.box {
  background-color: rgb(40, 200, 60);
  height: 100px;
}

Now we see our divs, but they're far from squares! They seem like one big blob of green! Let's set the width property to be 100px just like the height (so we have squares).

.box {
  background-color: rgb(40, 200, 60);
  height: 100px;
  width: 100px;
}

You may have noticed that the divs are now squares, but they're still on top of each other. What's that all about? Well, that's due to something called the display property. div elements have a default display property of block. The block display property, among other things, ensures that the object takes up the entire width of the screen. We don't want that. We want the boxes to take up just the allotted 100 pixels. We want a display property of inline-block. So, set the display property now.

.box {
  background-color: rgb(40, 200, 60);
  height: 100px;
  width: 100px;
  display: inline-block;
}

Things are looking far better! We've got some squares, and they're all side by side. Look at how far we've come! But things can always be better. For instance, notice the light blue border around the boxes in the example page. Let's give ours one like that too! To do this, we need to set three properties: border-width which sets the thickness of the border, border-color which is very clear about what it sets, and border-style which specifies what style of border (dotted, dashed, solid, etc). There is also, however, an easier way of doing this. There is a single command to set all three. That command is simply, border. The border property requires three values separated by spaces: the first of which is the border width, the second is the border style and the third is the border color. Go ahead and set the border width of our boxes to 4px, the border style to solid, and the border color to lightblue.

.box {
  background-color: rgb(40, 200, 60);
  height: 100px;
  width: 100px;
  display: inline-block;
  border: 4px solid lightblue;
}

The boxes look great, don't get me wrong, but there's just something about them that's not quite right. I think they're just too close together. Let's add a little bit of space between them. This property (space between objects) is called margin. Let's set the margin property to 10px now.

.box {
  background-color: rgb(40, 200, 60);
  height: 100px;
  width: 100px;
  display: inline-block;
  border: 4px solid lightblue;
  margin: 10px;
}

Okay, that's a little better, but things still feel a bit stuffy, don't you think? How about if we add a little space between the div boundary and the text within it the div? This property (the space within the object before another object can start) is called padding. Let's go ahead and set the padding property of our boxes to 20px.

.box {
  background-color: rgb(40, 200, 60);
  height: 100px;
  width: 100px;
  display: inline-block;
  border: 4px solid lightblue;
  margin: 10px;
  padding: 20px;
}

Our semi-final product looks great. But it's kind of strange the the boxes are so close to the edge of the page, right? Let's go ahead and make sure that there is at least 10 pixels on the top and bottom of the page (in addition to what's already there from the box margin), and 40 pixels on the left and right (in addition to what's already there from the box margins). When you've already done this, go ahead and check out the solution below.

body {
  background-color: rgb(40, 40, 40);
  padding: 10px 40px;
}
