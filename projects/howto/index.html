Gather all your things, children, because we're going on a little ride! Before we can submit any projects, we're going to have to have a plan of what this project should entail, right? Next, let's fire up our private email servers (we're looking at you, Hillary) and start a message to coderProjects@fvi.edu. This will be how we submit our project.

So you say you've got a project? Let's see it in action! As you probably already know, we have working versions of all our projects so that students can see what they're getting into and what the finished products should look like. We'd really appreciate it if you'd have a working version of yous too.

Once you've got a version of your project working and looking as you'd like, take all the files and zip them up. Take that zip file (containing all the required files to get your project working) and set it aside.

Don't forget to include in your zip file any of the following that might be pertinent:

  • HTML files
  • CSS files
  • JS files
  • Any images or other static files you might be using

Great! Now let's attach this to our ever improving email.

Let's start out these travels with a short summary of what this project is. What will we be creating in this project? Why is your project awesome? (Please be as self gratifying as possible here. Brag about your project. We don't mind. Shoot, we encourage it!).

What are some things that users should know or be familiar with before embarking on the voyage that is your proud project? Let's get them all arranged in a list and write them down. Try to make your requirements reasonable and realistic. If you don't actually need something for the project, don't include that thing as a prerequisite. Once you have a working list of prerequisites, go ahead and append them to your email.

We also would like to have a list of Learning Objectives. What should the student learn from doing this project? Where does it fall in the grand scheme of computation pedagogy? Once you've got that list, go ahead and append that to the email to.

Note: You should label these lists so as to avoid confusion and help the moderator read your suggested project.

Now we've arrived at our steps. Break your project down into smaller, digestible steps. What should be the task for the student at each step? What should be the hint (if there should be one for this step)? What should the code look like at this point?

Be sure to label which step is the last one. Any steps after the last will be taken as "extra credit". Don't shy away from the extra credit though! It's a great way to have one project that works for many different skill levels.

Then append each step (along with the step number) to your email.

Send off that email and you're done! Go away. Go outside. Play some basketball.
