In this project, we will get some experience using CSS frameworks by making some boxes with certain properties using MaterializeCSS. To see what the final version should look like, check out the finished product.

Let's set up an HTML file complete with a head and a body and a CSS file linked. You should already know how to do this, so this step should be very easy. If you are unsure of how to go about this, I suggest you try out the HTML basics code project or the Intro to CSS code project.

<!DOCTYPE html>

<html>
  <head>
    <link type='text/css' rel='stylesheet' href='/css/lecture3.css'>
  </head>
  <body>
  </body>
</html>

In order for our webiste to know that we're using the framework, we of course need to tell it that we are, right? Where and how do we link CSS stylesheets again? Inside the head tag, let's put a link tag with three attributes: type with a value of "text/css", rel with a value of "styleshee" and href with a value of the path to the style sheet. This time, however, our style sheet is not located on our own machine, but rather in a Content Delivery Network (CDN). Since we're using MaterializeCSS for this project, we'll use their CDN, which when this was written was here. You can just copy the link. So, let's go ahead and write that link with that as our href value. Make sure you write the link so that our file is called after the Materialize one. The reason for this is if we decide to overwrite anything in the Materialize CSS file, we can do so in our stylesheet and be assured that our version will persist because, as you recall, HTML is read from the top down.

<!DOCTYPE html>

<html>
  <head>
    <link type='text/css' rel='stylesheet' href='https://cdnjs.cloudflare.com/ajax/libs/materialize/0.97.5/css/materialize.min.css'>
    <link type='text/css' rel='stylesheet' href='/css/lecture3.css'>
  </head>
  <body>
  </body>
</html>

If you haven't already checked out the final version, please check it out before continuing so you know where things are headed and why we're doing the things we are doing. Being as how we've all seen the final project -- cough cough -- we can see that we have divs of a certain height and background color. Let's go ahead and set a CSS rule that gives all elements whose class property includes box a height of 225px.

.box {
  height: 225px;
}

Okay, so that was only one rule. Did you really want more though?

Whether you're using Bootstrap, Foundation, or Materialize as we are, you are probably going to need to understand the grid. What do we mean when we say "the grid"? Well, if you check out the final version of this project, you'll notice that the boxes stay the same width relative to the screen regardless of how big or small you make the window. Well, in a nutshell, that's one of the things the grid does for you.

So, you can think of webpages in these frameworks as being organized in 12 columns. If one wanted something to take up 6 of those columns (so half of the page), one would just have to specify that this is a 6 column element, however one does that in the framework he or she is using. In Bootstrap, this is done by adding the class "col-xs-6" to the element. In Materialize, however, this is done by adding two classes "col" and "s6".

At this point, you might be wondering what the "s" and the "xs" are all about. Well, sometimes you want your site to be arranged differently depeneding on the screen size. With that in mind, these frameworks come with a responsive grid. You can specify that on a very small screen, the element should be considered a 6 column element, but on a medium screen (or larger), it should be a 2 column element. This is done (in Materialize) by having three classes "col s6 m2". Similarly if an element has classes "col s6 m4 l2", then on a small screen it would be 6 columns, medium it would be 4 columns and on a large screen it would be 2 columns.

This may have been a lot to absorb in such a short while. With that in mind, we have a whole project to help these things soak in! So let's get started!

Let's make that first row of blue and green rectangles. In your HTML body add the following code:

<div class='row'>
  <div class='col s3 blue box'></div>
  <div class='col s3 green box'></div>
  <div class='col s3 blue box'></div>
  <div class='col s3 green box'></div>
</div>

Let's check out what this code does and then we'll analyze it. Firstly, you should notice that each of these elements is about 225 pixels high. That's because they all have the box class. Furthermore, you probably noticed that the box with class blue is blue, and the box with class green is green. There's not too much to say about that.

You should notice that there are four columns of equal width on the screen. This is because they all have the classes col and s3. Since 3*4=12, we know that four columns of 3 will be the entire width of the page. Hence, these are all 1/4 of the page each.

Now you might have noticed that we put a div with class row as the parent of all of our columns. This is actually important for our rows to act properly. The moral of the story, make sure you have all of your rows in side of a div with class row.

Now you try it out... Make a row of four equal width columns (that fills up the width of the screen). To do this, you have to figure out how many columns each element has to take up to make the total equal to 12.

Hint: 4 * 3 = 12, so if we want 3 columns, each has to have a width of 4.

<div class='row'>
  <div class='col s3 blue box'></div>
  <div class='col s3 green box'></div>
  <div class='col s3 blue box'></div>
  <div class='col s3 green box'></div>
</div>

Sometimes you don't really want page to be full width, but rather you want it to have some sort of a gutter on the sides. These frameworks generally make it pretty easy to do that from within the grid system! All you have to do is have your row (the div with class row) inside of a div with class container. Let's go ahead and make a row with three equal width columns inside of a container.

<div class='container'>
  <div class='row'>
    <div class='col s4 green box'></div>
    <div class='col s4 blue box'></div>
    <div class='col s4 green box'></div>
  </div>
</div>

From now on in this project everything will be within this container element even if it is not specified.

Sometimes you want to have parts of your website sectioned off. As in, you might have one part dedicated to your company's excellent customer service record and some of the other perks of your company over others, and another section for your pricing. You probably want to have a bit of vertical space between these sections. To achieve this, all you have to do is place each section inside of a div with class section. Easy, right? That's kind of why we use these frameworks.

Go ahead and put three more rows in a section. Each row should have a different arrangement of columns. Try to get inventive, but remember, the total number of columns on the screen is 12! What happens if the total inside the row is greater than 12? Less than 12? Figure this out!

<div class='section'>
  <div class='row'>
    <div class='col s3 green box'></div>
    <div class='col s2 blue box'></div>
    <div class='col s3 green box'></div>
    <div class='col s4 blue box'></div>
  </div>
  <div class='row'>
    <div class='col s6 blue box'></div>
    <div class='col s3 green box'></div>
    <div class='col s3 blue box'></div>
  </div>
  <div class='row'>
    <div class='col s5 green box'></div>
    <div class='col s1 blue box'></div>
    <div class='col s2 green box'></div>
    <div class='col s4 blue box'></div>
  </div>
</div>
