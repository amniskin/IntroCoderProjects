In this code project, we will be making the classic decision game, "odds and evens". If you're unfamiliar with this game, crawl out from that rock you've been living under and check out the wikipedia article and come back. Before we continue, however, we should also make sure that we have all the supplies we'll need. We have a picture of someone holding out one finger, two fingers, and for the A students, we have a picture of a cheating.

We don't want to get too fancy with it -- we don't even have a working program yet. Let's leave the serious styling for later once it's functional. For now, let's just make:

  • A span inside the 'wins' area with a special id (this will be where we keep the number of wins)
  • A similar thing for losses
  • A spot for us to show the user's choice (one finger or two)
  • A spot for us to show the computer's choice

Step 3: Getting user input!

Now we're about ready to make some functions!

We're going to need a way to collect the data. By that we mean that we'll need to find out if the user has chosen odds or evens, one finger or two finger. We'll need to "find out" how many fingers the computer chooses to throw down.

Let's start with getting the user's input. Here is where we introduce the functions alert and prompt. alert is a function that takes one argument and makes a popup window with that same string. Go ahead and test it out by typing alert("this is a test"); into the javascript console. Continue testing it with other strings, string concatination and variables, etc. until you feel comfortable enough to move on.

Once you're familiar with how alert works prompt should be a breeze. prompt is also a function that takes one string argument and makes a popup box displaying that string, but this time it also has an input box and the function returns whatever the user types in that input box. To test this out, go ahead and type var str = prompt("this is the prompt text");, type hey there into the input field and press OK. Once you've done that, check out the value of str (it should be "hey there"). Mess around with this function a little bit to familiarize yourself with it because we'll be using that in this section.

Now we're ready to get started making our functions! We're going to need a function to get the user's parity choice. To do this, let's make a function that takes no arguments and returns whatever the user's parity choice is.

Hint: you may want to limit the user's input options in the prompt message. A good way to do this is to have your code read prompt("Please enter 1 for odd and 0 or even"); or something to that effect.

While we're at it, let's get a function to prompt the user for his or her choice of one or two fingers with the string "How many fingers would you like to throw down? Enter either 1 or 2."

. 
function getParity() {
  return prompt("What parity do you choose? Enter 0 for even and 1 for odd.");
}
function getNumFingers() {
  return prompt("How many fingers would you like to throw down? Enter either 1 or 2.");
}

Step 4: Getting the computer's choice!

We're also going to need a function that generates the computer's choice. For that we will use a random number generator to generate a random number between 0 and 1. To get a random parity out of this, we need only to notice that the probability of our random number generator generating a number greater than or equal to 0.5 is roughly the same as it generating one less than 0.5. So if we simply round the number that the random number generator gives us, it should be a random choice between 0 and 1. If we add one to that, we can have the computer randomly throw out 1 or 2 fingers!

Make a function called compChoice that takes no arguments and returns either 1 or 2 (chosen at random).

function compChoice() {
  var randInt = Math.round(Math.random());
  return randInt+1;
}

Step 5: Who won?!

We're going to need a function to determine whether or not the user won, of course, so let's make that one right now. This function needs to know which parity the user chose, how many fingers the user used and how many fingers the computer used. The function should then take those values and see if the total number of fingers thrown is odd or even, and check that against the user's parity choice.

Define a function userWon that takes 3 arguments, user, comp, and parity. This function should add user and comp together, find out if the sum is odd or even (think about using % 2), then return true if the sum is the same parity as the variable parity and false otherwise.

function userWon(user, comp, parity) {
  var sum = user + comp;
  var sumParity = sum % 2;
  return sumParity == parity;
}

Step 6: Let's show our hands!

So now we've got everything we need to make this program work, except we haven't shown the choices yet. It would be nice if the user could have a little visual of who chose how many fingers, wouldn't it? Let's get on that!

To make our lives easier, though, we're first going to figure out how to show one person's hands, then we'll worry about showing both. Make a function called showPlayerHand that takes two arguments as input. The first argument should be the html object where you want the hands to show (either of the two spots we made in step one to show the user's and computer's choices), and the second argument should be the number of fingers to show. The function should then change the inner html of the first argument to be the constant from step two that represents the number of fingers from the second argument. For example, if you named the two img constants (from step 2) one and two, and you named the two object constants userHand and compHand, then the function should be such that showPlayerHand(userHand, 2) replaces the inner HTML of userHand with two. To do this, you might need an if statement.

function showPlayerHand(player, numHands) {
  if(numHands == 1) {
    player.innerHTML = one;
  } else {
    player.innerHTML = two;
  }
}

Now that we've got a function to show one player's hand, let's make one called showHands to show both players' hands. This function should take two arguments (the number of fingers the user is throwing in and the number that the computer is throwing in), and show each player's respective hands.

showHands(user, comp) {

Step 7: Bringing it all together!

So now we've got functions to get the user's parity choice, the user's choice of how many fingers to throw in, generate the computer's choice, find out who won, and show everybody's choices. All that's left to do is bring it together.

Let's make a function called playGame that takes no arguments. When this function is called, it collects the user's parity choice and choice of number of fingers and saves them as variables. It then get's the computer's choice and saves that as a variable too. Once it's done that, it shows the two choices in their respective HTML objects. Then, it determines who won and if the user won, the playGame should:

  • increment the wins variable
  • replace the inner HTML of the wins object with the current number of wins
  • alert and congratulate the user of his or her win

If the user lost, on the other hand, the function should do a similar thing but with losses instead of wins.

function playGame() {
  var user = getNumFingers();
  var parity = getParity();
  var comp = compChoice();
  var winningUser = userWon(user, comp, parity);
  showHands(user, comp)
  if(winningUser) {
    numWins++;
    wins.innerHTML = numWins;
    alert("Congratulations! You've won! Play again?");
  }
}

Step 8: Let's make that button work!

It would be nice if the user didn't have to go into the console to get things working, wouldn't it? So let's make that happen with an event listener! Now, event listeners can listen for many types of events (mouse enter, mouseleave, mousedown, keydown, keyup, etc.) but we're going to only deal with the 'click' event here. The way you would add a general event listener would be to take your element and append .addEventListener(event, functionToRun) to it. So for instance, in our case adding a click handler would look like:

element.addEventListener('click', functionToRun);

Go ahead and grab that button element we created in step 1, and let's walk through attaching an event handler to it. Let's say your button ID is play, then we can attach a click handler to that button by writing

var playBtn = document.getElementById('play');
playBrn.addEventListener('click', functionToRun);

Since the function we want to run is our playGame function, let's go ahead and have the event listener do that function when clicked. It's important to note that we are not calling the function inside the event handler so we don't have the parenthesis you would normally use with functions. 

var btn = document.getElementById('play');
btn.addEvenetListener('click', playGame);

Step 9: Let's take it for a test drive!

Here is where you test your function completely. Now, you should have been testing each piece along the way, but it's always good to test the final product as well. Things you're looking for are:

  • Do the correct choices show for the user and the computer?
  • Do the correct results show? (e.g. if you won, does it show that you won?, etc)
  • Are the counters correct? (Wins and losses)
  • Are there any typos in any of the alerts, etc?
  • Does the distribution of computer choices seem to be random? By that one we really mean that if you try ten in a row and you still have not gotten both choices, the odds are that something has gone wrong with the computer choice function.
